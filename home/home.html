A big leap forward in software configuration

Control all config from one place

When config is bundled with code, changes are almost always made by developers, and the same modification needs to be applied in multiple locations.

Having all config centralized across â€“ everything, makes it easier to audit and control from one place. One thing to change, one thing to propagate, one thing to integrate to.

Never expose secure information again

In face of complexity, security is often disregarded. Passwords and other sensitive information always find their way into config files.

Applications written with ConfigHub in mind are much more secure. There is no on-disk data to exploit, find, exfiltrate. Everything is protected and in-memory. This makes it easy to pass audits.

Eliminate config duplication and errors

Many config attributes are the same between applications or across environments. Due to the inflexibility of configuration files they are often duplicated, creating a fertile ground for errors and a time drain for changes.

ConfigHub eliminates file and key-value duplication entirely through the use of context - and zero duplication means fewer errors and lots of time saved.

Dynamic modeling for any system topology

The ability to dynamically define the context of each configuration on a per-repository basis greatly simplifies configuration management and allows a ton of flexibility with configuration overrides.

ConfigHub has developed first-class support for dynamic configuration contexts, divorcing application instances from their requisite configuration data, and thus leads the way to generating a host-agnostic system topology.

ConfigHub is great when...

You manage a distributed system

ConfigHub will have the biggest impact on enterprises managing cloud/hybrid distributed systems. In these environments, teams manage many applications and instances. Removal of config dependencies greatly simplifies its management.

You are concerned about security

Security flaws on application level are most common, and security is only as strong as its weakest link. ConfigHub enables a security layer that does not burden the application code. End-points need not be aware how the data is hidden.

Config is increasingly complex

The sheer volume of applications reliant on multi environment/instance configuration, combined with infrastructure dynamism leads to unavoidable errors. ConfigHub removes these complexities entirely, and allows for infrastructure config feedback.

You don't want to learn another tool

ConfigHub is designed for low learning curve by using familiar concepts - repository, key/value pairs and config files. Context, a new concept to some, can be mastered within 5-10 minutes by any DevOps team member.

You want platform/language independence

ConfigHub is a polyglot with central control. So, no matter the shape, size, or form, your software always integrates the same way, on any platform, in any language.

You want all hands on deck

All DevOps can manage their slice of configuration. With access controls, security groups and token-based config push/pull, you can safely scope access for each team.
